# Pattern: Output contract and streams

## When to use this pattern

Use when changing output formatting, adding metadata, or emitting additional data.

## stdout

stdout emits **exactly one line**: the path to the output file when the run succeeds.

- Fetch mode: prints path to `context.md` in the current working directory.
- Plan mode: prints path to `plan.md` in the resolved output directory.

Do not print anything else to stdout.

## context.md format

Written by `cmd/contextual/main.go`. The file has two parts:

### 1. Preamble (fixed header)

```
This file was generated by the `contextual` tool. It contains fetched content
from one or more sources (Jira issues, Confluence pages, and/or web pages),
assembled to provide context for an AI agent or developer.

Each section below represents a single fetched item. Sections are separated by
a line of '=' characters and begin with a short metadata header (TYPE, ID, URL,
TITLE), followed by the item's full content in Markdown format.

The FIRST item is the primary subject — it corresponds directly to the first
argument passed on the command line. All subsequent items are supporting context
discovered via links, references, or related content within the primary item.

Total items: <n>
```

### 2. Item blocks (one per fetched item)

```
================================================================================
TYPE: <friendly name>
ID: <id>          ← omitted if empty (web URLs have no ID)
URL: <url>
TITLE: <title>
================================================================================

<content in Markdown>

```

Friendly type names (from `itemTypeName` in `main.go`):
- `Jira Issue`
- `Confluence Page`
- `Web Page`

## stderr

stderr receives log output from the `internal/logger.Logger`:
- In `ModeVerbose`: `[INFO]`/`[WARN]`/`[ERROR]` lines
- In `ModeProgress`: `loading context ` + `.`/`X` characters
- In `ModeSilent`: nothing

Do not mix logs into stdout.

## Overwrite protection

Before writing `context.md`, `planner.ConfirmOverwrite(path)` is called. If the file already exists, the user is prompted interactively. If they decline, the process prints `Aborted.` to stderr and exits non-zero.

## Compatibility guidance

Assume downstream tooling may parse:
- the separator line (`=` × 80)
- the header keys (`TYPE:`, `ID:`, `URL:`, `TITLE:`)

If you add new headers, append them after `TITLE:` for backwards compatibility.
